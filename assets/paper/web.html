Q1. AWS Elastic Beanstalk — Definition, Advantages, Deployment Models, and Steps

Definition:
 Elastic Beanstalk is an AWS Platform as a Service (PaaS) that automates deployment, scaling, and management of web applications and services. Developers upload code, and Elastic Beanstalk handles the infrastructure (EC2, Load Balancer, Auto Scaling, and Monitoring).

Advantages:

  1. Simplified deployment: No manual setup of servers or load balancers.
  2. Automatic scaling: Adjusts capacity based on demand.
  3. Managed environment: Handles updates, patches, and monitoring automatically.
  4. Multi-language support: Supports Java, Node.js, Python, PHP, .NET, Go, Ruby, and Docker.
  5. Integration: Works seamlessly with other AWS services (RDS, S3, CloudWatch).
  6. Cost-efficient: Pay only for the resources used.

Deployment Models:

  1. Single-instance environment:
  • Used for development or testing.
  • One EC2 instance without load balancing.
  1. Load-balanced, auto-scaling environment:
  • Used for production.
  • Distributes traffic across instances and scales automatically.
  1. Custom VPC environment:
  • Runs within a Virtual Private Cloud for enhanced security and custom networking.

Steps to Deploy a Scalable Application using Elastic Beanstalk:

  1. Package the application: Prepare code as a ZIP or WAR file.
  2. Create a new application: In AWS Management Console, select Elastic Beanstalk → Create New Application.
  3. Choose platform: Select runtime (e.g., Node.js, Java, Python).
  4. Configure environment: Select load-balanced, auto-scaling configuration for scalability.
  5. Upload and deploy: Upload the code bundle and deploy it.
  6. Monitor and manage: Use CloudWatch for metrics, Elastic Beanstalk dashboard for health checks, and RDS if needed for the database.

Conclusion:
 Elastic Beanstalk offers a managed, scalable, and cost-effective deployment method for web applications, minimizing infrastructure management effort.

Q2. CloudFormation and CloudWatch

AWS CloudFormation — Infrastructure as Code

Definition:
 AWS CloudFormation automates creation and management of AWS resources using templates written in JSON or YAML. It provides Infrastructure as Code (IaC) to deploy consistent environments.

Purpose:

  • Define and provision entire infrastructure (EC2, RDS, S3, IAM, etc.) through templates.
  • Avoid manual configuration errors and improve consistency.
  • Simplify rollback and updates through versioned templates.

Advantages:

  1. Automates infrastructure setup.
  2. Supports repeatable deployments.
  3. Provides dependency management between resources.
  4. Enables easy rollback on failure.
  5. Integrates with CI/CD pipelines.

Example:
 A CloudFormation template can launch a VPC, EC2 instance, and RDS database together automatically.

Amazon CloudWatch — Monitoring and Metrics

Definition:
 Amazon CloudWatch is a monitoring and observability service for AWS resources and applications. It collects operational data like CPU usage, memory, latency, and network traffic.

Key Features:

  1. Metrics: Tracks performance data (e.g., EC2 CPU utilization, S3 requests).
  2. Alarms: Triggers alerts or automated actions (e.g., auto-scaling).
  3. Logs: Stores and analyzes application and system logs.
  4. Dashboards: Displays real-time monitoring data.
  5. Integration: Works with Auto Scaling, SNS, and Lambda for automation.

Benefits:

  • Real-time visibility into performance and availability.
  • Enables proactive resource management.
  • Supports automated scaling and troubleshooting.

Example:
 If an EC2 instance crosses 80% CPU usage, CloudWatch can trigger an alarm to auto-scale more instances.

Conclusion:
 CloudFormation simplifies infrastructure creation, while CloudWatch ensures performance monitoring and operational efficiency.

Q3. AWS Messaging Services — SNS, SQS, SES

Overview:
 AWS Messaging Services allow communication and integration between distributed applications, ensuring reliability and scalability in cloud systems.

1. Amazon SNS (Simple Notification Service):

Purpose:
 A publish-subscribe (Pub/Sub) service for sending notifications to multiple subscribers.

Features:

  • Publishers send messages to “topics.”
  • Subscribers receive messages via Email, SMS, HTTP, or Lambda.
  • Supports event-driven architectures.
  • Integrated with CloudWatch and Lambda.

Use Cases:

  • Application alerts and notifications.
  • Real-time system updates.
  • Event broadcasting to multiple endpoints.

2. Amazon SQS (Simple Queue Service):

Purpose:
 A message queuing service that decouples application components.

Features:

  • Messages are sent to queues and processed asynchronously.
  • Ensures reliable delivery even during system failures.
  • Supports standard and FIFO queues.
  • Integrates with SNS and Lambda.

Use Cases:

  • Asynchronous job processing.
  • Task scheduling.
  • Buffering between microservices.

3. Amazon SES (Simple Email Service):

Purpose:
 A cloud-based email sending service for transactional, notification, and marketing emails.

Features:

  • High deliverability using AWS infrastructure.
  • Supports bulk and personalized emails.
  • Provides feedback loops (delivery, bounce, complaint notifications).
  • Integrates with Lambda and CloudWatch for automation.

Use Cases:

  • Sending order confirmations or password resets.
  • Newsletter or promotional campaigns.

Q4. AWS Security Management – IAM, KMS, and Data Security (At Rest & In Motion)

1. AWS Identity and Access Management (IAM)

Definition:
 IAM is a centralized service for managing user access and permissions to AWS resources. It ensures that only authorized users or systems can perform specific operations.

Core Components:

  • Users: Individual identities with login credentials.
  • Groups: Collections of users with shared permissions.
  • Roles: Temporary credentials for AWS services or applications.
  • Policies: JSON documents defining access permissions.

Key Features:

  1. Fine-grained access control (resource-level and action-level).
  2. Multi-Factor Authentication (MFA) for enhanced security.
  3. Role-based access for temporary or automated access.
  4. Centralized management via the AWS Console, CLI, or API.

Benefits:

  • Enforces least privilege principle.
  • Helps meet compliance requirements.
  • Protects against unauthorized data access.

2. AWS Key Management Service (KMS)

Definition:
 KMS is a managed encryption key service that allows users to create, control, and use cryptographic keys to protect AWS data.

How It Works:

  • KMS uses Customer Master Keys (CMKs) stored in Hardware Security Modules (HSMs).
  • AWS services (S3, EBS, RDS) integrate with KMS for automatic encryption/decryption.
  • Keys can be managed manually or automatically rotated.

Features:

  1. Centralized key management with audit trails.
  2. Fine-grained key usage permissions through IAM.
  3. Automatic encryption of stored data.
  4. Supports envelope encryption (data keys encrypted by master keys).

Example:
 An S3 bucket can automatically encrypt uploaded files using a KMS key (SSE-KMS).

UNIT 4: AWS Storage

Q5. Amazon S3 – Buckets, Objects, Storage Classes, and Static Website Hosting

1. Amazon S3 Basics

Definition:
 S3 (Simple Storage Service) provides object-based storage for any type of data such as files, images, videos, and backups.

Core Concepts:

  • Bucket: Logical container for data (unique name per region).
  • Object: The stored file and its metadata.
  • Key: The unique name for each object in a bucket.

Features:

  1. Unlimited storage capacity.
  2. 99.999999999% durability and high availability.
  3. Access control via IAM and bucket policies.
  4. Lifecycle management for automatic data transfer to lower-cost tiers.

3. Hosting Static Website on S3

Steps:

  1. Create an S3 Bucket: Name it as your domain (e.g., example.com).
  2. Upload Files: Add HTML, CSS, JS files.
  3. Enable Static Website Hosting:

Q6. Amazon EBS – Role, Snapshots vs Backups

1. Amazon Elastic Block Store (EBS)

Definition:
 EBS provides block-level storage volumes for use with EC2 instances. It acts like a hard disk attached to a virtual machine.

Features:

  1. Persistent storage that remains even after EC2 shutdown.
  2. Supports SSD and HDD options (gp3, io2, st1, sc1).
  3. High performance for transactional workloads (databases, OS).
  4. Automatic replication within an Availability Zone for durability.
  5. Supports encryption via KMS.

Use Cases:

  • Databases and file systems.
  • Boot volumes for EC2.
  • Big data and analytics workloads.

Q7. Amazon Glacier – Features, Retrieval Process, and Comparison with Other Storage Services

1. Definition

Amazon Glacier (now part of Amazon S3 Glacier) is a low-cost cloud storage service designed for long-term archival and data backup. It is ideal for data that is rarely accessed but must be retained for compliance or audit purposes.

2. Key Features

  1. Extremely low cost: Designed for archival data with minimal retrieval needs.
  2. High durability: 99.999999999% (11 nines) durability across multiple Availability Zones.
  3. Secure encryption: Data automatically encrypted using AWS KMS keys.
  4. Scalable storage: No capacity planning needed; scales automatically.
  5. Lifecycle integration: Can be configured in S3 lifecycle rules to automatically transition old data to Glacier.
  6. Retrieval options:
  • Expedited (1–5 minutes)
  • Standard (3–5 hours)
  • Bulk (5–12 hours)

3. Retrieval Process

  • When an object stored in Glacier is requested, a retrieval job is initiated.
  • Once retrieval is complete, data is temporarily stored in S3 for download.
  • Access is via the AWS Console, CLI, or SDKs.

Example:
 A company stores monthly backups in S3 Glacier. When an old backup is required, it issues a retrieval request, waits for completion, and then downloads the file.

Q8. AWS Database Services – Amazon DynamoDB and RDS (MySQL)

1. Amazon DynamoDB

Definition:
 DynamoDB is a fully managed NoSQL database service that provides high performance and scalability for applications that require low-latency access.

Key Features:

  1. Serverless: No need to manage infrastructure or servers.
  2. Scalable: Automatically scales throughput and storage.
  3. Fast performance: Millisecond response time for reads/writes.
  4. Flexible schema: Stores data as key-value or document (JSON-like) structures.
  5. Secure: Supports encryption at rest (KMS) and in transit (TLS).
  6. Global tables: Multi-region replication for high availability.
  7. Backup & Restore: Point-in-time recovery and on-demand backups.

Use Cases:
 E-commerce carts, gaming leaderboards, IoT telemetry data, and session management.

2. Amazon RDS (Relational Database Service)

Definition:
 RDS is a managed relational database service that automates administrative tasks such as backups, patching, and scaling.

Supported Engines:
 MySQL, PostgreSQL, MariaDB, Oracle, SQL Server, and Aurora.

Launching and Managing RDS Instance (MySQL):

Steps:

  1. Open RDS Console: Choose “Create Database.”
  2. Select Engine: Choose MySQL.
  3. Choose Deployment Model: Standard or Multi-AZ.
  4. Configure Settings: DB name, username, password, instance type, and allocated storage.
  5. Connectivity: Select VPC, subnet, and public access options.
  6. Storage and Backup: Enable automatic backups and encryption.
  7. Launch Instance: AWS provisions and starts the DB automatically.
  8. Access Database: Connect via endpoint using MySQL Workbench or CLI.

Advantages:

  • Automated backups and failover.
  • Scalable compute and storage.
  • Integrated monitoring via CloudWatch.
