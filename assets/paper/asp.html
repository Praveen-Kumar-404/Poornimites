Q1. What are the different file types used in an ASP.NET project?

ASP.NET projects consist of multiple file types, each handling a specific part of the web application. Overview of all primary categories:

1. Page Files

.aspx

  • Main Web Form page.
  • Contains HTML + ASP.NET server controls.
    Example:     Home.aspx

.aspx.cs / .aspx.vb

  • Code-behind file.
  • Contains C# or VB logic for the .aspx page.
    Example:     Home.aspx.cs

.ascx

  • User Control file.
  • Reusable UI block.
    Example:     Header.ascx

2. Configuration Files

Web.config

  • XML-based configuration file
  • Stores connection strings, authentication settings, session settings.

Machine.config

  • System-level configuration file for all ASP.NET applications on the server.

3. Master Page Files

.master

  • Defines a common layout for multiple .aspx pages.
  • Contains placeholders for content.

4. Class and Logic Files

.cs / .vb

  • Standalone class files for code reuse.
  • Located in App_Code or project root.

5. Global Application File

Global.asax

  • Application-level file.
  • Contains event handlers:
  • Application_Start
  • Session_Start
  • Application_Error
  • Application_End

6. Resource Files

.resx

  • Stores strings, images, culture-specific resources.

7. Data Files

.xml

  • Stores structured data or configuration.

.mdf

  • SQL Server database file used in local database projects.

8. Assembly Files

.dll

  • Compiled code stored in the bin folder.
  • Contains reusable classes.

9. Miscellaneous

.css – Stylesheets

.js – JavaScript files

.config – Additional configuration files

.ashx – HTTP handler file

Q2. Explain any three commonly used directives with examples.

ASP.NET directives control page-level configuration and behavior. They appear at the top of .aspx, .ascx, or .master files.

1. Page Directive

Purpose: Defines settings for an ASP.NET page.

Syntax:

<%@ Page Language="C#" AutoEventWireup="true" CodeFile="Home.aspx.cs" Inherits="Home" %>

Key attributes:

  • Language – C# or VB
  • CodeFile – Path to code-behind file
  • Inherits – Class name linked to code-behind
  • AutoEventWireup – Automatically binds events like Page_Load

Usage:
Controls compilation, inheritance, and basic page parameters.

2. Import Directive

Purpose: Imports a namespace for use inside the page.

Syntax:

<%@ Import Namespace="System.Data" %>

Usage:
Allows page code to use classes without writing full namespace paths.

Example:

DataTable dt = new DataTable();  

Without Import, full name would be required:

System.Data.DataTable dt = new System.Data.DataTable();

3. Register Directive

Purpose: Registers User Controls or custom controls so they can be used on a page.

Syntax:

<%@ Register TagPrefix="uc" TagName="Header" Src="~/Header.ascx" %>

Usage:
After registration, you can use the control:

<uc:Header runat="server" id="Header1" />

4. (Optional) Master Directive

If needed as the fourth option.

Purpose: Used in .master files to define master page behavior.

Example:

<%@ Master Language="C#" %>

Q3. What is an HTTP Handler in ASP.NET? How do you create and use an HTTP Handler application?

Definition

An HTTP Handler is a low-level request processor in ASP.NET that handles incoming HTTP requests.
Every request sent to the server is ultimately handled by a class that implements IHttpHandler.

Examples of built-in handlers:

  • .aspx → PageHandlerFactory
  • .asmx → WebServiceHandlerFactory
  • .ashx → GenericHandler

If you want full control over a request (like returning images, text, file content), you create a custom handler.

Why use an HTTP Handler

  • Fast and lightweight.
  • Used for tasks where Web Forms and MVC overhead is unnecessary.
  • Ideal for:
  • Serving images dynamically
  • Handling file downloads
  • Generating custom responses (text, JSON, XML)
  • Implementing mini-APIs

How to Create a Custom HTTP Handler

Step 1: Create a .ashx File

Example: ImageHandler.ashx

The file looks like:

<%@ WebHandler Language="C#" Class="ImageHandler" %>

using System;

using System.Web;

public class ImageHandler : IHttpHandler

{

    public void ProcessRequest(HttpContext context)

    {

        context.Response.ContentType = "text/plain";

        context.Response.Write("Hello from HTTP Handler");

    }

    public bool IsReusable

    {

        get { return false; }

    }

}

Step 2: Write the Response

Inside ProcessRequest(), you write what the handler should output.

Examples:

context.Response.Write("Generated text");

or serve an image:

context.Response.ContentType = "image/png";

context.Response.WriteFile("~/images/logo.png");

Step 3: Use the Handler

Access it using the browser:

https://yoursite.com/ImageHandler.ashx

The output is whatever you wrote in ProcessRequest().

Q4. What are Standard Controls in ASP.NET Web Forms? Explain the use of Label, TextBox, Button, and DropDownList controls.

Standard Controls are built-in Web Form controls provided by ASP.NET to build user interfaces.
They are server-side controls, meaning they run on the server and support events (click, change, etc.).

Below are the four required controls:

1. Label Control

Purpose

Displays non-editable text on a page.

Properties

  • Text – text to display
  • ForeColor – text color

Example

<asp:Label ID="Label1" runat="server" Text="Enter Name:"></asp:Label>

Used when you need titles, headings, or to show messages.

2. TextBox Control

Purpose

Allows the user to input text.

Modes

  • Single-line
  • Multi-line (TextMode="MultiLine")
  • Password field (TextMode="Password")

Example

<asp:TextBox ID="TextBox1" runat="server"></asp:TextBox>

Used for forms: login, search box, comments, etc.

3. Button Control

Purpose

Triggers an action on click (server-side event).

Event

OnClick — runs a function in code-behind.

Example

<asp:Button ID="Button1" runat="server" Text="Submit" OnClick="Button1_Click" />

Code-behind:

protected void Button1_Click(object sender, EventArgs e)

{

    Label1.Text = "Button clicked!";

}

Used for submitting forms and triggering actions.

4. DropDownList Control

Purpose

Provides a drop-down menu to select one item.

Key Properties

  • Items — set of options
  • SelectedValue — selected item

Example

<asp:DropDownList ID="DropDownList1" runat="server">

    <asp:ListItem Text="India" Value="IN"></asp:ListItem>

    <asp:ListItem Text="USA" Value="US"></asp:ListItem>

</asp:DropDownList>

Used for country selection, categories, roles, etc.

Q5. Explain the use of the TreeView class and TreeView control in ASP.NET with a suitable example.

Purpose

The TreeView control displays information in a hierarchical (parent–child) structure.
It is used when data must be shown in expandable/collapsible format.

Examples:

  • Website navigation
  • Folder structure
  • Categories and sub-categories

ASP.NET provides:

  • TreeView class → the underlying class in System.Web.UI.WebControls
  • TreeView control → visual server control used inside .aspx pages

Features of TreeView

  • Expand/Collapse nodes
  • Parent and child nodes
  • Navigation using node links
  • Auto-formatting and styling
  • Supports data binding (XML, SiteMap)
  • Supports events like SelectedNodeChanged

TreeView Structure

The control contains:

  • TreeNode → single node
  • ChildNodes → collection of sub-nodes
  • Text → label shown on screen
  • NavigateUrl → link address

Example (Manual TreeView)

ASPX

<asp:TreeView ID="TreeView1" runat="server">

    <Nodes>

        <asp:TreeNode Text="Subjects">

            <asp:TreeNode Text="Computer Science"></asp:TreeNode>

            <asp:TreeNode Text="Mathematics"></asp:TreeNode>

            <asp:TreeNode Text="Physics"></asp:TreeNode>

        </asp:TreeNode>

        <asp:TreeNode Text="Departments">

            <asp:TreeNode Text="IT"></asp:TreeNode>

            <asp:TreeNode Text="HR"></asp:TreeNode>

            <asp:TreeNode Text="Finance"></asp:TreeNode>

        </asp:TreeNode>

    </Nodes>

</asp:TreeView>

This produces a collapsible structure:

Subjects

   ├── Computer Science

   ├── Mathematics

   └── Physics

Departments

   ├── IT

   ├── HR

   └── Finance

Example (Using SiteMapDataSource)

<asp:TreeView ID="TreeView1" runat="server" DataSourceID="SiteMapDataSource1"></asp:TreeView>

<asp:SiteMapDataSource ID="SiteMapDataSource1" runat="server" />

TreeView automatically builds its structure using Web.sitemap.

Q6. What are Validation Controls in ASP.NET? Explain any three types of validation controls with examples.

Definition

Validation Controls are built-in ASP.NET controls used to enforce input rules, ensuring the user enters valid data before form submission.

They run:

  • On client side (JavaScript)
  • On server side

All validation controls run automatically when Page.IsValid is checked.

Common Types of Validation Controls

Below are three easy, high-scoring ones.

1. RequiredFieldValidator

Purpose: Ensures the user does not leave a field empty.

ASPX Example

Name: <asp:TextBox ID="TextBox1" runat="server"></asp:TextBox>

<asp:RequiredFieldValidator

    ControlToValidate="TextBox1"

    runat="server"

    ErrorMessage="Name is required">

</asp:RequiredFieldValidator>

2. RangeValidator

Purpose: Checks whether input falls within a specific numeric or date range.

Example

Age: <asp:TextBox ID="TextBox2" runat="server"></asp:TextBox>

<asp:RangeValidator

    ControlToValidate="TextBox2"

    runat="server"

    MinimumValue="18"

    MaximumValue="60"

    Type="Integer"

    ErrorMessage="Age must be 18–60">

</asp:RangeValidator>

3. RegularExpressionValidator

Purpose: Validates format using a regex.
Used for emails, phone numbers, pin codes.

Example

Email: <asp:TextBox ID="TextBox3" runat="server"></asp:TextBox>

<asp:RegularExpressionValidator

    ControlToValidate="TextBox3"

    runat="server"

    ValidationExpression="\w+@\w+\.\w+"

    ErrorMessage="Invalid Email Format">

</asp:RegularExpressionValidator>

Other validators (for completeness)

  • CompareValidator → compares two fields
  • CustomValidator → user-defined logic
  • ValidationSummary → shows all errors in one box

Q7. Explain the use and features of the GridView Control in ASP.NET.

Definition

The GridView control is a powerful data-bound control used to display data in a table format.
It supports:

  • Displaying
  • Sorting
  • Paging
  • Editing
  • Deleting
  • Selecting rows

It is one of the most commonly used controls in ASP.NET Web Forms.

Key Features of GridView

1. Auto-generates columns

If AutoGenerateColumns="true", GridView automatically creates columns based on data source fields.

2. Supports paging

Breaks data into pages:

AllowPaging="true"

3. Supports sorting

Enables clicking on column headers to sort:

AllowSorting="true"

4. Supports editing and deleting

Enables built-in "Edit", "Update", "Cancel", "Delete" buttons without writing much code.

5. Template Columns

Allows custom layout using:

<asp:TemplateField>

6. Data binding

Works with:

  • SqlDataSource
  • ObjectDataSource
  • List, DataTable, Dataset
  • LINQ data

7. Styling

Supports:

  • Header style
  • Row style
  • Alternating row colors

Example of GridView

ASPX

<asp:GridView ID="GridView1" runat="server"

    AutoGenerateColumns="true"

    AllowPaging="true"

    AllowSorting="true">

</asp:GridView>

<asp:SqlDataSource

    ID="SqlDataSource1"

    runat="server"

    ConnectionString="<%$ ConnectionStrings:DBCon %>"

    SelectCommand="SELECT * FROM Students">

</asp:SqlDataSource>

Result

A table like:

StudentID

Name

Age

Course

With paging and sorting links automatically added.

Q8. Discuss the working of the Chart Control in ASP.NET. Mention its key properties.

Definition

The Chart control in ASP.NET is used to display graphical charts such as:

  • Line
  • Bar
  • Pie
  • Area
  • Column
  • Bubble
  • Pyramid

Located in System.Web.UI.DataVisualization.Charting.

How the Chart Control Works

  1. You place the Chart control on a Web Form.
  2. You define chart Series (data points).
  3. You define Chart Areas (plotting regions).
  4. You configure appearance using properties.
  5. Finally, you bind data (manual or through a data source).

Main Components of a Chart

1. Series

A Series represents a set of data points.

Example:

Chart1.Series["Sales"].Points.AddXY("Jan", 100);

2. Chart Area

Defines the plotting area where the chart is drawn.

Example:

Chart1.ChartAreas.Add("MainArea");

3. Legends

Displays labels explaining what each series represents.

4. Titles

Used to display chart heading.

Key Properties of Chart Control

1. ChartType

Defines the type of chart.

Examples:

Chart1.Series["Sales"].ChartType = SeriesChartType.Column;

Chart1.Series["Sales"].ChartType = SeriesChartType.Line;

2. DataSource

Binds data to a chart.

Chart1.DataSource = ds;

3. Palette

Allows selection of color themes for the chart.

Chart1.Palette = ChartColorPalette.Bright;

4. Titles

Add chart titles.

Chart1.Titles.Add("Monthly Sales");

5. Legends

Used to show explanations for series.

Example Chart Control in ASPX

<asp:Chart ID="Chart1" runat="server">

    <Series>

        <asp:Series Name="Sales" ChartType="Column">

            <Points>

                <asp:DataPoint AxisLabel="Jan" YValues="200" />

                <asp:DataPoint AxisLabel="Feb" YValues="350" />

                <asp:DataPoint AxisLabel="Mar" YValues="150" />

            </Points>

        </asp:Series>

    </Series>

    <ChartAreas>

        <asp:ChartArea Name="MainArea" />

    </ChartAreas>

</asp:Chart>

Q9. Write a short note on the importance of Controllers in handling user input and responses.

Definition

In an ASP.NET MVC application, a Controller is a class responsible for:

  • Receiving user requests
  • Processing the request
  • Interacting with data (Models)
  • Returning a response (usually a View or JSON)

Controllers are the entry point of every MVC request.

Role and Importance of Controllers

1. Handle user input (HTTP Requests)

Controllers receive requests through action methods.
Example:

public IActionResult Index()

{

    return View();

}

When a user opens /Home/Index, the Index() action handles it.

2. Act as a bridge between Model and View

Controllers:

  • Fetch data from Models
  • Prepare data
  • Send data to Views

Flow:

User → Controller → Model → Controller → View

3. Manage application logic

Controllers contain decision-making code:

  • Validations
  • Conditions
  • Selecting which View to show
  • Redirecting

Example:

if (id == null)

    return RedirectToAction("Error");

4. Return different types of responses

Controllers can return:

  • View() → HTML page
  • Json() → JSON data
  • File() → files
  • Redirect() → redirect to another action
  • Content() → plain text

5. Maintain separation of concerns

Controllers separate:

  • UI logic (View)
  • Data logic (Model)
  • Request handling (Controller)

This improves readability and maintainability.

Example Controller

public class StudentController : Controller

{

    public IActionResult Details(int id)

    {

        var student = db.Students.Find(id);

        return View(student);

    }

}

Controller receives ID → fetches student → sends to View.

Q10. Discuss the advantages of using ASP.NET MVC for web application development.

1. Separation of Concerns (MVC Architecture)

MVC divides application into:

  • Model → data & business logic
  • View → UI
  • Controller → request handling

This makes code structured, clean, and easier to maintain.

2. Support for Test-Driven Development (TDD)

Controllers and models are simple classes, making them easy to test using unit tests.
Views can also be tested separately.

3. No ViewState or PostBack overhead

Unlike Web Forms, MVC does not use:

  • ViewState
  • Postback model

Result:

  • Faster performance
  • Lightweight pages

4. Full control over HTML

ASP.NET MVC gives developers complete control over:

  • HTML markup
  • CSS
  • JavaScript

No auto-generated heavy markup like Web Forms.

5. Better for RESTful applications

MVC maps URLs to controller actions using Routing.
This makes it ideal for:

  • REST APIs
  • clean URLs
  • SEO-friendly websites

Example of clean URL:

/products/details/10

6. Easier integration with client-side frameworks

MVC supports modern frontend libraries:

  • Angular
  • React
  • Vue
  • jQuery

Views are simple Razor templates, making integration straightforward.

7. Built-in features

ASP.NET MVC includes:

  • Model binding
  • Data annotations
  • Layouts & partial views
  • Filters (Authorization, Caching, Logging)

These reduce repetitive coding.

8. Extensible and customizable

MVC pipeline is open and can be extended:

  • Custom routes
  • Custom filters
  • Dependency injection
  • Custom model binders
