1. Insertion in AVL Tree

An AVL tree is a self-balancing binary search tree where the height difference (balance factor) between left and right subtrees of any node is –1, 0, or +1.

Balance Factor (BF)=height(left)−height(right)\text{Balance Factor (BF)} = \text{height(left)} - \text{height(right)}Balance Factor (BF)=height(left)−height(right)

If this condition is violated after insertion, rotations are performed to restore balance.

Steps for Insertion

  1. Insert the new node using standard BST rules (left < root < right).
  2. Update balance factors while backtracking toward the root.
  3. If BF of a node becomes ±2, perform appropriate rotation to rebalance.

Rotation Cases

There are four rotation types to fix imbalance:

Imbalance Type

Condition

Rotation Type

LL

Node inserted in left subtree of left child

Single Right Rotation

RR

Node inserted in right subtree of right child

Single Left Rotation

LR

Node inserted in right subtree of left child

Left Rotation on left child, then Right Rotation on parent

RL

Node inserted in left subtree of right child

Right Rotation on right child, then Left Rotation on parent

Example

Insert sequence: 30, 20, 10

  • 30 root → insert 20 (left child) → insert 10 (left of 20).
  • Node 30 becomes unbalanced (BF = +2, left-heavy).
  • Perform Right Rotation at 30 → 20 becomes root.

Time Complexity: O(log⁡n)O(\log n)O(logn)
Space Complexity: O(1)O(1)O(1)
Tree remains balanced after each insertion.

2. 2–3 Tree Insertion

A 2–3 Tree is a perfectly balanced multiway search tree where:

  • A 2-node has one key and two children.
  • A 3-node has two keys and three children.
  • All leaves are at the same depth.

Rules

  • Keys inside a node are sorted in ascending order.
  • Subtrees maintain order:
    Left < first key < middle < second key < right.

Insertion Steps

  1. Find the leaf position where new key should go.
  2. Insert the key into the node (now may have 2 or 3 keys, sorted).
  3. If the node becomes overfull (3 keys):
  • Split it into two 2-nodes.
  • Promote the middle key to the parent.
  1. If the parent also overflows, repeat the split upward.
  2. If the root splits, the promoted key becomes new root, increasing tree height by 1.

Example

Insert keys: 10, 20, 30

  • Insert 10 → forms 2-node [10].
  • Insert 20 → [10, 20].
  • Insert 30 → node overflows → split into [10] and [30], promote 20 → 20 becomes root.

Properties:

  • Tree remains balanced.
  • Every insertion causes at most one split per level.
  • Time Complexity: O(log⁡n)O(\log n)O(logn)

3. B-Tree

A B-Tree of order mmm is a self-balancing search tree where each node can have up to m children and m–1 keys.
Used in databases and file systems to minimize disk I/O.

Properties

  1. Each node (except root) has between ⌈m/2⌉\lceil m/2 \rceil⌈m/2⌉ and mmm children.
  2. Root has at least two children unless it is a leaf.
  3. All leaves appear on the same level.
  4. Keys in a node are sorted, and child pointers partition key ranges.

Insertion Steps

  1. Find the leaf node for the key (normal search).
  2. Insert the key into the node in sorted order.
  3. If the node exceeds maximum keys (m−1)(m - 1)(m−1)**:
  • Split the node around the middle key.
  • Promote the middle key to parent.
  • Adjust pointers accordingly.
  1. If the parent also overflows, repeat splitting upward.
  2. If the root splits, new root is created with the promoted key.

Example (Order 3 B-Tree)

Each node can have max 2 keys and 3 children.
Insert keys: 10, 20, 30, 40, 50

  • [10, 20]
  • Insert 30 → overflow → split → 20 promoted.
  • [20] root; children [10] and [30].
  • Insert 40, 50 → 30’s node overflows → split → promote 40.
  • Final structure: [20, 40]; children [10], [30], [50].

Complexities

  • Search: O(log⁡n)O(\log n)O(logn)
  • Insertion: O(log⁡n)O(\log n)O(logn)
  • Deletion: O(log⁡n)O(\log n)O(logn)
  • Space: O(n)O(n)O(n)

Key Property

All operations maintain logarithmic height and minimize disk access.
B-trees are ideal for large datasets stored on secondary storage.

4. Deletion and Search in 2–3 Tree

Search in a 2–3 Tree

A 2–3 tree search is similar to binary search but extended for multi-key nodes.
Every internal node contains one or two keys, which divide data ranges into two or three subtrees.

Algorithm:

  1. Start at the root.
  2. If the search key matches any key in the current node → found.
  3. Else, follow the appropriate branch:
  • If key < smallest key → move to left child.
  • If key is between two keys → move to middle child.
  • If key > largest key → move to right child.
  1. Continue until found or until reaching a leaf.
    If leaf reached and key not found →     key not present.

Time Complexity: O(log⁡n)O(\log n)O(logn) because the tree is perfectly balanced.
Space Complexity: O(1)O(1)O(1).
All root-to-leaf paths are equal in length, ensuring consistent search performance.

Deletion in a 2–3 Tree

Goal: Remove a key while keeping all leaves at equal depth and nodes valid (1 or 2 keys).

Cases:

Case 1: Key in Leaf Node

  • If the leaf has two keys, simply delete the key.
    (Example:     [10, 20] → delete 20 → [10])
  • If the leaf has one key, deletion may cause underflow, requiring borrowing or merging.

Case 2: Key in Internal Node

  • Replace the key with its in-order predecessor or successor (from leaf level).
  • Then delete that replacement key from the leaf.
    This reduces deletion to the leaf case.

Balancing Steps (for underflow):

  1. Borrowing:
  • If a sibling has two keys, borrow one key via the parent.
  • The parent’s separating key moves down, and sibling key moves up to parent.
  1. Merging:
  • If sibling has one key, merge the current node with its sibling.
  • The parent’s separating key joins the merged node.
  • If parent now has no keys, propagate merge upward.
  1. If Root becomes Empty:
  • The single child becomes the new root.
  • Tree height decreases by one.

Time Complexity: O(log⁡n)O(\log n)O(logn).
All rebalancing operations occur within logarithmic depth.

5. Red-Black Tree Insertion

A Red-Black Tree is a binary search tree (BST) with extra coloring rules that enforce near-perfect balance.

Properties

  1. Every node is red or black.
  2. The root is always black.
  3. Every NIL (null) pointer is considered a black leaf.
  4. Red nodes cannot have red children (no two consecutive reds).
  5. Every path from a node to descendant NILs has the same number of black nodes.

These properties guarantee a maximum height of 2log⁡2(n+1)2 \log_2(n+1)2log2​(n+1), ensuring O(log⁡n)O(\log n)O(logn) search, insert, and delete.

Insertion Steps

  1. Standard BST insertion: Insert node as red leaf.
  2. Check for violations (two consecutive red nodes).
    Let:
  • N = newly inserted node
  • P = parent
  • U = uncle
  • G = grandparent

Cases:

  • Case 1: Parent is black → Tree valid, stop.
  • Case 2: Parent and uncle are red →
    Recolor parent and uncle to black, grandparent to red.
    Move pointer to grandparent and recheck.
  • Case 3: Parent is red and uncle is black → Rotation required.
  • Left-Left (LL): Right rotate at grandparent.
  • Right-Right (RR): Left rotate at grandparent.
  • Left-Right (LR): Left rotate at parent, then right rotate at grandparent.
  • Right-Left (RL): Right rotate at parent, then left rotate at grandparent.
    After rotation, recolor nodes (new root black).

Final Step: Root must always be black.

Time Complexity: O(log⁡n)O(\log n)O(logn).
Insertion triggers at most two rotations and constant recoloring.

6. Red-Black Tree Deletion

Goal: Remove a node and restore red-black properties.

Steps

  1. Perform standard BST deletion.
  2. If the deleted node or its replacement was red, simply remove (no violation).
  3. If the deleted node was black, it creates a double black situation that violates property 5.

Let:

  • X = node with double black (can be NIL)
  • S = sibling of X
  • P = parent

Fixing Double Black Cases:

Case

Condition

Action

1

Sibling red

Recolor sibling black, parent red, rotate toward X, proceed to next case

2

Sibling black and both children black

Recolor sibling red, move double black up to parent

3

Perform rotation and recoloring to remove double black

4

Final Step: Root is always recolored black at end.

Time Complexity: O(log⁡n)O(\log n)O(logn)
Rebalancing Cost: Constant number of rotations and recolors.

  • 2–3 Tree Deletion: Borrowing and merging ensure full balance after removal.
  • Red-Black Tree Insertion/Deletion: Color and rotation-based balancing maintain logarithmic operations without full tree restructuring.

7. Topological Sorting (DFS and BFS Based)

Definition:
 Topological sorting of a Directed Acyclic Graph (DAG) is a linear ordering of its vertices such that for every directed edge u→vu \to vu→v, vertex uuu appears before vvv in the ordering.
It applies only to DAGs.

A. DFS-Based Topological Sort

Idea:
 Perform DFS and add each vertex to a stack after visiting all its descendants (post-order).

Steps:

  1. Mark all vertices as unvisited.
  2. For each unvisited vertex, call DFS.
  3. In DFS:
  • Mark the current vertex as visited.
  • Recursively visit all unvisited adjacent vertices.
  • Push the vertex onto a stack when all its neighbors are processed.
  1. After DFS completes, pop elements from the stack to get the topological order.

Example:

Graph:

5 → 0 ← 4

↓         ↑

2 → 3 → 1

DFS traversal order: 5, 2, 3, 1, 4, 0
Topological order: 4, 5, 2, 3, 1, 0

Time Complexity: O(V + E)

B. BFS-Based (Kahn’s Algorithm)

Idea:
 Uses in-degree (number of incoming edges) to determine which nodes can be placed next.

Steps:

  1. Calculate in-degree for all vertices.
  2. Add all vertices with in-degree = 0 to a queue.
  3. While queue not empty:
  • Dequeue a vertex and add it to the topological order.
  • For each outgoing edge from that vertex, reduce the in-degree of the destination vertex by 1.
  • If any vertex’s in-degree becomes 0, enqueue it.
  1. Continue until all vertices are processed.

Time Complexity: O(V + E)

Use Case: Task scheduling, dependency resolution, course prerequisite order.

8. Dijkstra’s Algorithm (Single Source Shortest Path)

Definition:
 Dijkstra’s algorithm finds the shortest path from a source vertex to all other vertices in a weighted graph with non-negative edge weights.

Steps:

  1. Initialize:
  • Distance of source vertex = 0.
  • Distance of all other vertices = ∞.
  • Create a priority queue (min-heap) storing pairs (vertex, distance).
  1. While queue not empty:
  • Extract vertex u with the smallest distance.
  • For each adjacent vertex v of u:
  • If distance[u] + weight(u, v) < distance[v], update distance[v].
  • Push (v, distance[v]) into the queue.
  1. Repeat until all vertices are processed.

10. Directed Graph

Definition:
 A Directed Graph (Digraph) is a graph where each edge has a direction, represented as an ordered pair (u,v)(u, v)(u,v), meaning there is an edge from vertex u to vertex v, but not necessarily from v to u.

Components:

  • Vertices (V): Nodes in the graph.
  • Edges (E): Ordered pairs showing direction, e.g., (A,B)(A, B)(A,B) means A → B.
  • In-degree: Number of incoming edges to a vertex.
  • Out-degree: Number of outgoing edges from a vertex.
  • Path: Sequence of vertices connected by directed edges.
  • Cycle: Path where the first and last vertices are the same.
  • Acyclic Graph: Directed graph with no cycles (DAG).

Representation:

  1. Adjacency Matrix:
     Square matrix where A[i][j] = 1 if edge from i → j exists, else 0.
    Space: O(V2)O(V^2)O(V2)
  2. Adjacency List:
     Each vertex stores a list of vertices it points to.
    Space: O(V+E)O(V + E)O(V+E)

11. Topological Sorting (DFS and BFS Based)

Definition:
 Topological sorting is a linear ordering of vertices in a Directed Acyclic Graph (DAG) such that for every directed edge u→vu \to vu→v, vertex u appears before v in the order.

Condition: Works only for DAGs (no cycles).

A. DFS-Based Topological Sorting

Concept:
 Perform Depth First Search and push the node to a stack after exploring all its neighbors.
When DFS completes, pop the stack to get the topological order.

Algorithm Steps:

  1. Mark all vertices as unvisited.
  2. For each unvisited vertex, call DFS.
  3. In DFS:
  • Mark the node as visited.
  • For each adjacent node, call DFS recursively.
  • After visiting all neighbors, push the node into the stack.
  1. Pop all nodes from the stack to get the topological order.

Example:

Graph:

5 → 0

4 → 0

4 → 1

2 → 3

3 → 1

DFS Order → [4, 5, 2, 3, 1, 0]
Topological Order → [4, 5, 2, 3, 1, 0]

Time Complexity: O(V + E)

B. BFS-Based Topological Sorting (Kahn’s Algorithm)

Concept:
 Use in-degree (number of incoming edges) to determine which nodes can come next.

Algorithm Steps:

  1. Calculate in-degree for each vertex.
  2. Add all vertices with in-degree = 0 into a queue.
  3. While queue is not empty:
  • Remove a vertex from the queue and add it to the result.
  • For each of its outgoing edges, decrease the in-degree of the destination vertex.
  • If any vertex’s in-degree becomes 0, enqueue it.
  1. If all vertices are processed, result is the topological order.
