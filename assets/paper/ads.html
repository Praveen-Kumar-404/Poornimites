Advance Data Structure

Hashing — Unit 1

1. Linear Probing

Technique used in open addressing when a collision occurs.

  • If the hash index h(k) is occupied, probe sequentially:
    h(k), h(k)+1, h(k)+2, ... (mod m)
  • Primary clustering occurs: long blocks of filled slots form, increasing search time.
  • Insertion: find first empty slot in the linear sequence.
  • Search: follow same probe sequence until key found or empty slot reached.
  • Deletion: use “deleted marker” to avoid breaking probe chain.
  • Cost: average successful search ≈ O(1/(1-α)), α = load factor.

2. Quadratic Probing

Probing with quadratic increments to reduce clustering.

  • Probe sequence:
    h(k) + 1², h(k) + 2², h(k) + 3², ... (mod m)
  • Reduces primary clustering, but secondary clustering still occurs (keys with same hash generate same sequence).
  • Works only if table size m is prime and load factor is low (≤ 0.5).
  • No guarantee of finding an empty slot unless conditions on m and probing function satisfied.

3. Double Hashing

Best open-addressing method.

  • Second hash function defines the step size:
    h1(k) + i * h2(k)
     with h2(k) non-zero.
  • Produces wide probe distribution → reduces both primary and secondary clustering.
  • Conditions:
  • h2(k) should be independent of h1(k)
  • Usually: h2(k) = 1 + (k mod (m-1))
  • More uniform, better average-case performance.

4. Rehashing

Technique to maintain performance when load factor becomes too high.

  • When α exceeds threshold (e.g., 0.75), create new table of larger size (often 2×).
  • Recompute hash for every key and insert into new table.
  • Cost of rehash: O(n) but amortized cost per operation remains O(1) because rehash happens rarely.
  • Prevents clustering and performance degradation.

5. Universal Hashing

Family of hash functions chosen randomly at runtime.
Goal: avoid worst-case input crafted to cause collisions.

  • Universal family H ensures:
    For any two distinct keys     x ≠ y,
    Pr(h(x) = h(y)) ≤ 1/m
  • Prevents adversarial collision patterns.
  • Construction example (Carter-Wegman):
    For prime     p > m:
    h_{a,b}(k) = ((a*k + b) mod p) mod m
     where a ∈ {1,…,p−1}, b ∈ {0,…,p−1} chosen randomly.
  • Used in cryptography, randomized algorithms, secure hash tables.

Unit 2 - Priority Queues (Heaps)

1. Binary Heap — Structure Property

A binary heap is a complete binary tree:

  • All levels completely filled except possibly the last.
  • Last level filled from left to right.
  • Stored efficiently as an array; index formulas:
  • Parent: i/2
  • Left child: 2i
  • Right child: 2i + 1

This structure ensures height = O(log n).

2. Binary Heap — Order Property

For min-heap:

  • Key at each node ≤ keys of its children.

For max-heap:

  • Key at each node ≥ children.

Root always contains minimum (min-heap) or maximum (max-heap).
Invariant maintained during all operations.

3. Binary Heap — Operations

All operations rely on restoring structure + order property.

Insertion

  • Add new key at next empty position (bottom-right).
  • Restore order using percolate-up (bubble-up):
    Compare with parent; swap until heap-order holds.
  • Complexity: O(log n).

Extract-Min / Extract-Max

  • Remove root.
  • Move last element to root.
  • Restore order using percolate-down (heapify-down).
  • Complexity: O(log n).

Decrease-Key / Increase-Key

  • Update value at index.
  • Restore order by percolate-up or percolate-down depending on the change.

Build-Heap

Two methods:

  • Insert each element one-by-one → O(n log n)
  • Floyd’s method (optimal):
  • Start heapify from last non-leaf down to root
  • Complexity: O(n)

Heap supports priority queue operations efficiently.

4. Binomial Queue — Structure

A binomial queue is a collection of binomial trees.
A binomial tree B_k:

  • Degree k
  • Formed by linking two B_{k-1} trees
  • Has 2^k nodes
  • Root degree = k
  • Children arranged from highest to lowest degree
    Important property:
  • Queue contains 0 or 1 tree of each degree (like binary representation of n).

5. Binomial Queue — Operations

Binomial queues are good when merge is a high-frequency operation.

Merge (Union) — Fundamental Operation

  • Combine two queues like binary addition:
  • Compare roots of trees of equal degree
  • Link trees when degrees match
  • Complexity: O(log n).

Insert

  • Create a single-node tree (B_0)
  • Merge with existing queue
  • Complexity: O(log n).

Find-Min

  • Scan root list to find min root
  • Complexity: O(log n)
     (There are at most log n roots).

Extract-Min

  • Steps:
  1. Find the root with minimum key
  2. Remove its tree
  3. Reverse its children’s order to form a separate binomial queue
  4. Merge with the original queue minus the removed tree
  • Complexity: O(log n).

6. Binomial Queue — Implementation Notes

  • Represented using linked structures (each node has pointers to first child and sibling).
  • Must maintain correct linking when merging or extracting.
  • Ensures amortized bounds for priority queue operations.

Unit 3 - Trees

1. AVL Trees

Self-balancing binary search tree maintaining height difference ≤ 1.

AVL Rotations

Rotations restore balance after insertion or deletion.
Balance factor = height(left) − height(right) ∈ {−1, 0, +1}.

Four cases

  1. LL Rotation (Right Rotation)
  • Heavy on left-left side.
  • Rotate right around unbalanced node.
  1. RR Rotation (Left Rotation)
  • Heavy on right-right side.
  • Rotate left around unbalanced node.
  1. LR Rotation (Left-Right)
  • Inserted in left child’s right subtree.
  • First rotate left (child), then rotate right.
  1. RL Rotation (Right-Left)
  • Inserted in right child’s left subtree.
  • First rotate right (child), then rotate left.

Each rotation restores height balance in O(1) time.

AVL Insertion

  1. Insert like BST.
  2. Recompute heights bottom-up.
  3. Identify first unbalanced node.
  4. Apply correct rotation case (LL, RR, LR, RL).
  5. Tree height remains O(log n).

AVL ensures guaranteed logarithmic time for search/insert/delete.

2. Multi-way Search Trees — 2-3 Trees

Each internal node has:

  • Either 2 children & 1 key (2-node), or
  • 3 children & 2 keys (3-node).

Always perfectly balanced.

Search

  • Compare key with 1 or 2 keys inside node.
  • Move to correct child.
  • Height = O(log n) and same for all leaves.

Insert

  1. Insert key in leaf node.
  2. If leaf becomes 3-key node (overflow), split:
  • Middle key moves up to parent
  • Remaining two keys form two separate nodes
  1. Splitting may propagate to root.
  2. If root splits → new root is created, height increases by 1.

Always maintains balanced height.

Delete

  1. Delete from leaf; if internal, swap with predecessor/successor.
  2. If node becomes underfull, merge or borrow from siblings.
  3. Propagates upward until balance restored.

Deletion is more complex but still O(log n).

3. Red-Black Trees

Binary search tree with node coloring ensuring balanced height.

Properties

  1. Each node is red or black.
  2. Root is black.
  3. No red node has a red child.
  4. Every path from a node to leaves has same number of black nodes.
    These rules guarantee height ≤     2 log(n).

Insertion (Simple Cases)

Steps:

  1. Insert as red node (BST insert).
  2. If parent is black → done.
  3. If parent is red → violation occurs.
    Simple cases include:
  • Recolor when uncle is red
  • Rotate when uncle is black
  1. Final rotation + recoloring fixes structure.

Insertion uses 1–2 rotations max.

Deletion (Simple Cases)

Full deletion is long; exam focuses on simple versions:

  1. Delete red leaf → safe, no fix needed.
  2. If removing black node, double-black issue arises.
    Simple cases include:
  • Sibling red → rotation + recolor
  • Sibling black with black children → recolor and move up
    Other complex cases usually skipped in exams.

Unit 4 - Graph Algorithms

1. Topological Sort

Applies only to Directed Acyclic Graphs (DAGs).

Goal

Produce a linear order of vertices such that every directed edge
u → v appears with u before v.

Two Methods

Method 1: DFS-based

  1. Run DFS on every unvisited vertex.
  2. After finishing a vertex, push it into a stack.
  3. Final topological order = reverse of the finishing order.

Method 2: Kahn’s Algorithm (Indegree Method)

  1. Compute indegree of all vertices.
  2. Put all vertices with indegree 0 in queue.
  3. Pop, output, and reduce indegree of neighbors.
  4. Add new indegree-0 vertices to queue.

Key Points

  • Detects cycle: if not all nodes output → graph has a cycle.
  • Time: O(V + E).

2. Single Source Shortest Path — Dijkstra’s Algorithm

Works only when all edge weights ≥ 0.

Core Idea

Use greedy strategy with priority queue.
Maintain dist[] array where dist[v] = shortest distance from source.

Steps

  1. Initialize all distances = ∞ except source = 0.
  2. Use min-priority queue.
  3. Extract-min vertex u.

Relax all edges (u, v):

if dist[v] > dist[u] + w(u,v)

    dist[v] = dist[u] + w(u,v)

  2. Repeat until queue empty.

Complexity

  • With binary heap: O((V + E) log V).

Correctness

  • Non-negative edges ensure once a vertex distance is finalized, it is optimal.

3. Single Source Shortest Path — Bellman–Ford Algorithm

Works for graphs with negative weights and detects negative cycles.

Core Idea

Relax all edges repeatedly.

Steps

  1. Initialize distances same as Dijkstra.

Repeat V-1 times:
Relax all edges (u, v):

dist[v] = min(dist[v], dist[u] + w(u,v))

  2. After V-1 iterations, run one more iteration:
  • If any edge can still be relaxed → negative cycle exists.

Features

  • Handles negative edges.
  • Detects negative cycles.
  • Time: O(VE).

4. All-Pairs Shortest Paths — Floyd–Warshall Algorithm

Dynamic programming method. Works for positive and negative weights. No negative cycles allowed.

Idea

DP over intermediate vertices.
dist[i][j] = shortest path from i to j using vertices from set {1…k} as intermediates.

Recurrence

For each k from 1 to V:

dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])

Complexity

  • Time: O(V³)
  • Space: O(V²)
     Used mainly for dense graphs or small V.

Unit 5 - Disjoint Sets and String Matching

1. Disjoint Set Data Structure (Union–Find)

Used to maintain a collection of non-overlapping sets and support fast merging and querying.

Core Operations

  • Find(x): returns the representative (root) of the set containing x.
  • Union(x, y): merges the sets containing x and y.

Representation

Each set is represented as a tree, where each node points to its parent.
Root of each tree is the representative.

2. Union by Rank / Size

Optimizations to keep trees shallow.

Union by Rank

  • Each tree has a rank (approx. height).
  • When merging two trees:
  • Attach smaller rank tree under larger rank tree.
  • If ranks equal, pick one as root and increase its rank by 1.

Union by Size

  • Attach smaller-sized tree under larger-sized tree.

Effect

Prevents tall trees → improves Find() speed.

3. Path Compression

Applied during Find(x).

Mechanism

After finding the root of x, make every node on the search path directly point to the root.

Example

If path: x → p → q → root,
after compression: x → root, p → root, q → root.

Result

Trees become extremely flat.

Total Complexity

With union-by-rank + path compression:

  • Amortized per operation: O(α(n)),
    where     α(n) = inverse Ackermann function (grows slower than log, effectively constant).

4. Rabin–Karp String Matching

Uses rolling hash to find pattern P in text T.

Idea

Instead of comparing strings directly, compare hash values.

Steps

  1. Compute hash of pattern P.
  2. Compute hash of first window of T.

Slide window by one position using rolling hash formula:

hash_next = (d*(hash_current - T[i]*h) + T[i+m]) mod q

  1.  where:
  • d = alphabet size
  • h = d^(m−1)
  • q = large prime
  1. If hashes match → verify characters to avoid collision.

Features

  • Average Case: O(n + m)
  • Worst Case: O(nm) due to collisions
  • Good for multiple pattern matching and plagiarism detection.

5. KMP (Knuth–Morris–Pratt) Algorithm

Avoids re-checking matched characters by precomputing pattern structure.

Core Idea

Use LPS array (Longest Prefix Suffix) for pattern P.

LPS Array

For each index i, lps[i] = length of longest proper prefix that is also a suffix for P[0..i].

KMP Steps

Step 1: Preprocess Pattern

Compute LPS array in O(m) time.

Step 2: Search

Scan text T:

  • If characters match: move both pointers.
  • If mismatch after some matches:
  • Move pattern pointer to lps[j−1],
  • Do not shift text pointer back.

Time Complexity

  • Preprocessing: O(m)
  • Search: O(n)
  • Total: O(n + m)
     Always worst-case linear.

Advantages

  • Deterministic.
  • No backtracking in text.
  • Far more predictable than Rabin–Karp.
