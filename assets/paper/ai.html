1. What is Artificial Intelligence and what are Intelligent Agents?

Artificial Intelligence (AI)
 AI is the study and creation of systems that can perform tasks requiring human-level intelligence. These tasks include learning, reasoning, problem-solving, perception, and decision-making.
AI focuses on building machines that behave rationally by choosing the best possible action to achieve goals.

Intelligent Agent (IA)
 An intelligent agent is an autonomous entity that:

  • Perceives its environment through sensors
  • Acts on the environment using actuators
  • Processes information to choose rational actions
  • Maximizes a performance measure

Agent Components:

  • Sensors: Cameras, microphones, input signals
  • Actuators: Motors, displays, outputs
  • Agent Program: Logic deciding what action to take
  • Agent Architecture: Physical or software framework supporting the program

Types of Agents:

  • Simple Reflex Agents
  • Model-Based Reflex Agents
  • Goal-Based Agents
  • Utility-Based Agents
  • Learning Agents

2. What are the different approaches of AI?

AI has four classical approaches, each defining how intelligence is measured.

(a) Acting Humanly (Turing Test Approach)
 A machine is considered intelligent if it behaves like a human and can convince an evaluator that it is human. Requires:

  • Natural language processing
  • Knowledge representation
  • Reasoning
  • Learning

(b) Thinking Humanly (Cognitive Modeling)
 Focuses on replicating human thought processes. Cognitive scientists and AI researchers model:

  • How humans learn
  • How humans solve problems
  • How memory and reasoning work

(c) Thinking Rationally (Laws of Thought)
 Uses logic-based reasoning. The system derives correct conclusions from given premises.
Limitation: real-world situations are often uncertain or incomplete.

(d) Acting Rationally (Rational Agent Approach)
 Modern AI uses this approach.
A rational agent chooses the best possible action to achieve goals, considering available information and constraints.

3. How does AI solve problems using search? Explain problem formulation.

AI treats every problem as a search through a state space.

A search problem is defined by five elements:

  1. Initial State
     Starting point of the problem.
    Example: Taxi at location A.
  2. Actions (Operators)
     All possible actions from a state.
    Example: Move north, move south, pick up passenger.
  3. Transition Model
     Describes what happens after performing an action.
    Example: Move north from A → Go to B.
  4. Goal Test
     Determines whether the current state is the desired state.
    Example: Passenger dropped at destination.
  5. Path Cost Function
     A numerical cost assigned to a path.
    Example: Distance, time, or fuel consumed.
    Notation:
    g(n)=cost from initial state to node ng(n) = \text{cost from initial state to node } ng(n)=cost from initial state to node n

Search Process:

  • AI generates possible states using a search tree.
  • Explores states using search algorithms (BFS, DFS, A*, etc.).
  • Finds a path from initial state to goal state.

State Space = all possible configurations of the problem.

Solution = a sequence of actions leading to the goal.

4. Explain Uninformed Search techniques: BFS, DFS, Iterative Deepening, Bidirectional Search.

Uninformed search works without any heuristic or extra knowledge about the goal. It explores the state space blindly.

Breadth-First Search (BFS)

  • Explores nodes level by level starting from the root.
  • Uses a FIFO queue.
  • Complete: finds a solution if one exists.
  • Optimal: only when all step costs are equal.
  • Time/Space Complexity:
    O(bd)O(b^d)O(bd)
    where
    b=b =b= branching factor,
    d=d =d= depth of the goal.

Strength: Guarantees shortest path.
Weakness: Very high memory use.

Depth-First Search (DFS)

  • Explores deepest node first.
  • Uses stack (explicit stack or recursion).
  • Memory requirement is low.

Not complete in infinite-depth spaces.
Not optimal because it may find deeper costly paths first.

Complexity:

  • Time: O(bm)O(b^m)O(bm) where mmm is maximum depth
  • Space: O(b⋅m)O(b \cdot m)O(b⋅m)

Strengths:

  • Low memory
  • Works well on deep, single-solution problems

Weaknesses:

  • Can get stuck in infinite paths
  • Not guaranteed to find the best solution

Iterative Deepening Search (IDS)

Combination of BFS and DFS.

Process:

  1. Run DFS with depth limit = 0
  2. Increase limit to 1, run DFS
  3. Increase limit to 2, run DFS
  4. Continue until goal is found

Advantages:

  • Memory-efficient like DFS
  • Optimal like BFS (when all actions have same cost)

Complexity:

  • Time: O(bd)O(b^d)O(bd)
  • Space: O(bd)O(bd)O(bd)

Bidirectional Search

Runs two searches simultaneously:

  • One from the initial state → goal
  • One from the goal → initial state

They meet in the middle.

Complexity reduces drastically:

  • Each side explores roughly:
    bd/2b^{d/2}bd/2

instead of bdb^dbd.

Strength: Fastest uninformed search for many problems.
Requirement: Goal state must be explicitly known.

5. Explain Hill Climbing and its limitations.

Hill climbing is a local search algorithm that moves toward the direction of maximum improvement.

It evaluates neighbors of the current state and moves to the one with the highest value.

It does not store search history.

Types of Hill Climbing

  • Steepest-Ascent Hill Climbing
     Evaluate all neighbors, choose the best.
  • Simple Hill Climbing
     Evaluate neighbors one by one, take the first better move.
  • Random-Restart Hill Climbing
     Run hill climbing multiple times from random states.

Limitations (Major Problems)

  1. Local Maxima
     A peak that is lower than the global maximum.
    Algorithm gets stuck.
  2. Plateaus
     Area where many neighboring states have the same value.
    No direction for improvement → stuck.
  3. Ridges
     Narrow path where the best direction is neither straight uphill nor downhill.
    Algorithm oscillates or makes poor progress.
  4. Greedy Behavior
     Always moves upward; cannot move temporarily downward to reach a better global solution.

Conclusion:
 Hill climbing is fast but unreliable for complex, multi-peaked landscapes.

6. Explain Informed Search techniques: Heuristics, Greedy Search, A* Search, AO* Search.

Informed search uses heuristics to guide the search.

Heuristic (h(n))

A heuristic is an estimate of the cost from the current node nnn to the goal.

Properties:

  • Admissible if it never overestimates actual cost
  • Consistent if it satisfies triangle inequality:
    h(n)≤c(n,n′)+h(n′)h(n) \leq c(n,n') + h(n')h(n)≤c(n,n′)+h(n′)

Good heuristics drastically reduce search time.

Greedy Best-First Search

Selects the node with the lowest heuristic value h(n)h(n)h(n).

Characteristics:

  • Fastest informed search
  • Not optimal
  • Not complete in infinite spaces

Evaluation function:

f(n)=h(n)f(n) = h(n)f(n)=h(n)

Moves toward the "closest-looking" goal.

A* Search

A* uses both path cost and heuristic.

Evaluation function:

f(n)=g(n)+h(n)f(n) = g(n) + h(n)f(n)=g(n)+h(n)

Where:

  • g(n)g(n)g(n): cost from start to node nnn
  • h(n)h(n)h(n): heuristic cost to goal

A* is optimal when heuristic is:

  • Admissible
  • Consistent

Strength: Best balance between speed and accuracy.
Weakness: High memory usage.

AO* Search

Used in AND-OR graphs, where some nodes require solving multiple subproblems simultaneously (AND nodes).

Features:

  • Handles problems where one decision leads to multiple required branches
  • Expands only promising branches
  • Uses heuristics to decide optimal solution graph

Used in planning, reasoning, and expert systems.

7. What are Constraint Satisfaction Problems (CSP)?

A Constraint Satisfaction Problem is a problem defined by variables, domains, and constraints, where the goal is to assign values to all variables such that every constraint is satisfied.

Components

  1. Variables:
     Represent unknowns in the problem.
    Example: In a map-coloring problem → regions (A, B, C…).
  2. Domains:
     Possible values each variable can take.
    Example: Colors {Red, Green, Blue}.
  3. Constraints:
     Rules that restrict variable assignments.
    Example: Adjacent regions cannot have the same color.

Types of Constraints

  • Unary constraint: Involves one variable.
  • Binary constraint: Involves two variables.
  • Higher-order: Involves three or more variables.

Examples of CSP

  • Map coloring
  • Sudoku
  • Cryptarithmetic puzzles
  • Scheduling problems
  • Exam timetable generation

Solving Techniques

  1. Backtracking Search
     Depth-first assignment with backtracking when constraints break.
  2. Forward Checking
     When a value is assigned, eliminate conflicting values from neighbors.
  3. Arc Consistency (AC-3 Algorithm)
     Ensures that for every variable pair (X, Y), every value in X has a compatible value in Y.
  4. Heuristics
  • MRV (Minimum Remaining Values): Choose variable with least legal values.
  • LCV (Least Constraining Value): Choose value that rules out the fewest choices for others.

Why CSPs are powerful

CSP framework separates what the problem is from how to solve it.
Works efficiently for large search spaces with strong constraints.

1. Explain Game Playing in AI with the Minimax Algorithm.

Game playing in AI involves designing an agent that makes optimal decisions in competitive environments (two-player zero-sum games).

Characteristics of Game-Playing Problems

  • Adversarial: Two players compete (MAX vs MIN).
  • Deterministic: No randomness (chess, tic-tac-toe).
  • Perfect information: Both players know the entire state.

Minimax Algorithm

A decision-making algorithm that chooses a move by assuming:

  • MAX tries to maximize score.
  • MIN tries to minimize score.

Working

  1. Construct a game tree from the current board.
  2. Evaluate terminal states with a utility function.
  3. Back-propagate values:
  • MAX picks the maximum child value.
  • MIN picks the minimum child value.

Utility Example

  • Win = +1
  • Draw = 0
  • Loss = –1

Properties

  • Optimal: Gives best move if the opponent plays optimally.
  • Complete: If game tree is finite.
  • Expensive: Grows exponentially with depth.
    Complexity:
    O(bd)O(b^d)O(bd)
    where bbb = branching factor, ddd = depth of game tree.

Limitations

  • Cannot explore full game tree in complex games like chess.
  • Requires pruning or depth-limited search.

2. How does Alpha-Beta Pruning optimize Minimax?

Alpha-Beta pruning reduces the number of nodes explored in the minimax tree without affecting the result.

Key Idea

Stop exploring a branch when it cannot influence the final minimax decision.

Two Parameters

  • α (alpha): Best value MAX can guarantee so far.
  • β (beta): Best value MIN can guarantee so far.

Pruning Rule

If at any node:

α≥β\alpha \geq \betaα≥β

the branch is pruned (not explored).

Effect

  • Does not change the result (optimal remains optimal).
  • Cuts off unnecessary branches.
  • Allows deeper search in the same time.

Best-case Performance

If moves are evaluated in optimal order:

O(bd/2)O(b^{d/2})O(bd/2)

This is a massive improvement over minimax’s O(bd)O(b^d)O(bd).

Example

If MAX already has a move worth 8, and MIN finds a branch worse than 8 for MAX, MIN will never allow MAX to reach 8, so the branch is pruned.

Practical Use

Used in:

  • Chess engines
  • Checkers
  • Go (with modifications)
  • Any adversarial game with deterministic rules

1. Explain the Jug Problem in detail.

The Water Jug Problem is a classic AI search problem where the goal is to measure an exact quantity of water using two jugs of different capacities.
It is solved by exploring the state space using BFS, DFS, or other search methods.

Problem Structure

You are given:

  • Jug X with capacity A liters
  • Jug Y with capacity B liters
  • A target quantity C liters

State Representation

A state is written as:

(x,y)(x, y)(x,y)

where

  • xxx = current water in Jug X
  • yyy = current water in Jug Y

Operations

You can perform the following allowed actions:

  1. Fill a jug completely
  • Fill X → (A,y)(A, y)(A,y)
  • Fill Y → (x,B)(x, B)(x,B)
  1. Empty a jug
  • Empty X → (0,y)(0, y)(0,y)
  • Empty Y → (x,0)(x, 0)(x,0)
  1. Pour water from one jug to another
  • Pour from X to Y until X is empty or Y is full
  • Pour from Y to X until Y is empty or X is full

Example

Jugs: 4 L and 3 L
Goal: measure 2 L

Solution sequence using BFS:

  1. (0,0)
  2. (4,0) — Fill X
  3. (1,3) — Pour X → Y
  4. (1,0) — Empty Y
  5. (0,1) — Pour X → Y
  6. (4,1) — Fill X
  7. (2,3) — Pour X → Y → Goal reached

Why AI uses this problem

  • Teaches state-space search
  • Shows how BFS finds shortest sequence
  • Demonstrates problem formulation in AI

Q2(a) What is partial order planning? How does it differ from linear planning? Explain with an example.

Partial Order Planning (POP)

A planning technique where a plan is not strictly ordered; only necessary constraints are included.
Also called least commitment planning.

POP allows:

  • Actions to be unordered when the order doesn’t matter
  • Parallel execution of independent actions
  • Avoiding unnecessary orderings

Characteristics

  • Maintains a set of actions, ordering constraints, and causal links
  • Allows flexibility
  • Avoids committing to sequences early

Linear Planning

  • Every action is placed in a fixed sequence
  • Total order: A → B → C
  • No flexibility
  • Poor for parallel tasks

Q1(a) Illustrate the working of a Decision Tree in supervised learning. Mention one strength and one limitation.

A Decision Tree is a supervised machine learning model used for classification and regression.
It splits data into branches based on feature conditions until it reaches a final decision (leaf node).

Working of a Decision Tree

1. Select the Best Feature to Split the Data

The tree chooses the feature that provides the highest information gain or lowest Gini impurity.

Measures used:

  • Entropy
     Entropy=−∑pilog⁡2(pi)Entropy = -\sum p_i \log_2(p_i)Entropy=−∑pi​log2​(pi​)
  • Information Gain
     Measures reduction in uncertainty after a split.

2. Create a Decision Node

Node represents a test condition.
Example:
“Is Age < 30?”
“Yes” → left branch
“No” → right branch

3. Recursively Split the Dataset

Each subset is further split until:

  • Pure class labels
  • Maximum depth reached
  • Minimum samples left

4. Create Leaf Nodes

Leaf nodes store the final output (class label or numeric value).

Example

Predict whether a customer will buy a laptop.

Features:

  • Income
  • Age
  • Student status

Sample tree:

          Income?

         /         \

      High         Low

       |             \

   Age < 25?       Student?

Each test divides data until a clear decision appears.

Strength

  • Easy to understand, visualize, and interpret.

Limitation

  • Highly prone to overfitting if not pruned.

Q1(b) Demonstrate how a Support Vector Machine (SVM) separates data into classes with the help of an example.

A Support Vector Machine (SVM) is a supervised learning model that classifies data by finding the optimal separating boundary (hyperplane).

Key Idea

SVM finds the hyperplane that maximizes the margin between classes.

Hyperplane Equation

w⋅x+b=0w \cdot x + b = 0w⋅x+b=0

Margin

Distance between the hyperplane and the closest data points (support vectors).
SVM chooses the hyperplane with the largest margin, improving generalization.

Example

Two classes:

  • Red points
  • Blue points

SVM identifies the line that best separates them.
The closest points to the boundary are support vectors, which directly influence the hyperplane.

If the data is not linearly separable, SVM uses:

  • Kernel trick
  • Maps data to higher-dimensional space

Example kernels:

  • Linear
  • Polynomial
  • RBF (Gaussian)

Result

SVM creates a robust classifier that handles high-dimensional data effectively.

Introduction to Natural Language Processing (NLP)

Natural Language Processing (NLP) is a branch of Artificial Intelligence that enables machines to understand, interpret, and generate human language.
It combines:

  • Linguistics
  • Machine learning
  • Deep learning
  • Computational algorithms

NLP deals with tasks such as:

  • Speech recognition
  • Machine translation
  • Sentiment analysis
  • Chatbots
  • Text summarization
  • Question answering

Its core purpose is to allow computers to interact with humans using natural language.

Different Issues Involved in NLP

NLP faces several challenges due to the ambiguity, complexity, and variability of human languages.

1. Ambiguity

Words and sentences often have multiple meanings.

  • Lexical ambiguity:
     “Bank” → riverbank or financial institution
  • Syntactic ambiguity:
     “I saw the man with a telescope.”
  • Semantic ambiguity:
     Meaning missing or unclear

Ambiguity makes language interpretation difficult.

2. Morphological Issues

Words change form based on tense, number, gender, etc.
Example:
Run, running, ran → same root but different forms.

3. Syntactic Issues

Understanding sentence structure.
Example:
Subject–verb agreement, phrase ordering, grammar rules.

4. Semantic Issues

Understanding the meaning of words or sentences.
Examples:

  • Word sense disambiguation
  • Metaphors
  • Idioms
  • Hidden meaning

5. Pragmatic Issues

Understanding language in context.
Examples:

  • Tone
  • Intent
  • Social meaning
  • Sarcasm
  • Cultural references

Example:
“Can you pass the salt?” → request, not a question about ability.

6. World Knowledge Issues

For correct interpretation, a system needs background knowledge.
Example:
“John dropped the glass. It broke.”
System must know:

  • Glass is fragile
  • "It" refers to glass

7. Speech-related Issues

For spoken language:

  • Background noise
  • Accents
  • Speed variations
  • Mispronunciations

These issues make NLP a challenging domain requiring advanced models like RNNs, Transformers, and BERT.

Expert System

An Expert System is an AI software designed to mimic the decision-making ability of a human expert.

Components

  1. Knowledge Base
     Contains rules and facts
  2. Inference Engine
     Applies rules to reach conclusions
  3. User Interface
     Interaction layer

How It Works

  1. User enters a query
  2. Inference engine checks rules
  3. Matches facts and produces a decision
  4. Provides explanation

Features

  • Rule-based
  • Domain-specific
  • Uses logical reasoning

Applications

  • Medical diagnosis (MYCIN)
  • Troubleshooting systems
  • Legal advice
  • Credit risk evaluation

Expert systems were early powerful AI systems, though now supplemented by deep learning.

Robotics

Robotics is an AI-integrated field involving the design, construction, and operation of robots capable of performing tasks autonomously or semi-autonomously.

Core Components of Robotics

  1. Sensors
     Cameras, microphones, infrared sensors
  2. Actuators
     Motors, arms, wheels
  3. Control Systems
     Microcontrollers, CPUs
  4. AI Algorithms
     Planning, perception, NLP, computer vision

Role of AI in Robotics

  • Perception: Object detection, speech recognition
  • Localization & Mapping: SLAM techniques
  • Motion Planning: A*, RRT algorithms
  • Manipulation: Robotic arms and grippers
  • Decision-making: Reinforcement learning
  • Human–Robot Interaction: NLP for conversation

Applications of Robotics

  • Manufacturing (assembly lines)
  • Healthcare (surgical robots)
  • Agriculture (harvesting robots)
  • Defense (drones)
  • Home automation (cleaning robots)
  • Space exploration (Mars rovers)
