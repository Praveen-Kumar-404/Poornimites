Question 1: Propositional Logic and First-Order Logic — Differences and Examples

1. Propositional Logic (PL)

  • Definition:
     Deals with statements (propositions) that are either True (T) or False (F).
    Example:
  • P: It is raining
  • Q: I will take an umbrella
  • Statement: If it is raining, then I will take an umbrella.
     → Symbolic form: P → Q
  • Elements:
  • Propositions: Simple or compound statements.
  • Connectives: AND (∧), OR (∨), NOT (¬), IMPLIES (→), BICONDITIONAL (↔).
  • Truth Values: T or F only.
  • Limitation:
     Cannot express relationships or quantify over objects (e.g., “all,” “some”).

2. First-Order Logic (FOL)

  • Definition:
     Extends propositional logic by using predicates, quantifiers, and variables to express relationships between objects.
  • Components:
  • Constants: Specific objects (e.g., Alice, Bob)
  • Variables: Represent any object (e.g., x, y)
  • Predicates: Properties or relations (e.g., Loves(Alice, Bob))
  • Quantifiers:
  • ∀ (For all)
  • ∃ (There exists)
  • Example:
  • P(x): x is a cat
  • Q(x): x has a tail
  • “All cats have tails” → ∀x (P(x) → Q(x))

Question 2: Knowledge Base in Artificial Intelligence and Steps in Building It

Definition:
 A Knowledge Base (KB) is a repository of structured information that an AI system uses to reason, solve problems, and make intelligent decisions.

Components of a Knowledge Base:

  1. Facts:
     Statements about the world.
    Example: “Paris is the capital of France.”
  2. Rules:
     Logical conditions to infer new facts.
    Example: If it is raining, then the road is wet.
  3. Inference Engine:
     Applies reasoning (forward/backward chaining) to derive conclusions.
  4. Knowledge Acquisition:
     Gathering data from experts, databases, or documents.

Types of Knowledge:

  1. Declarative: Facts — what is true.
    Example: “The sky is blue.”
  2. Procedural: How to do things.
    Example: “How to drive a car.”
  3. Heuristic: Experience-based shortcuts.
    Example: “If traffic is heavy, take a shortcut.”

Steps in Building a Knowledge Base:

Step

Description

1. Knowledge Acquisition

Collect information from experts or data.

2. Knowledge Representation

Encode in a structured form (rules, semantic networks, frames).

3. Knowledge Validation

Verify accuracy and consistency.

4. Knowledge Inference

Use reasoning to generate new facts.

5. Knowledge Refinement

Continuously update and improve.

Example:

  • Fact: “All birds can fly.”
  • Fact: “Sparrow is a bird.”
  • Inference Engine: → “Sparrow can fly.”

Applications:
 Expert systems, chatbots, medical diagnosis, robotics, decision support systems.

Question 3: Situation Calculus — Representation of Actions and Effects in Dynamic Environments

Definition:
 Situation Calculus is a logical language used to represent and reason about actions, changes, and their effects over time in dynamic systems.

Purpose:
 To describe how actions affect the world and what remains unchanged after each action.
Used in robotics, automated planning, and reasoning systems.

Main Components:

  1. Situation (S):
     A state of the world at a point in time.
    Example: S₀ = initial situation.
  2. Action (A):
     Operation that changes the world.
    Example: Open(Door)
  3. Fluents:
     Properties that can vary across situations.
    Example: DoorOpen(Door1, S)

Representation:

  • Performing action A in situation S gives a new situation: do(A, S)
  • Example:
  • Action: Open(Door1)
  • Fluent: DoorOpen(Door1, S)
  • Precondition: Poss(Open(Door1), S) ↔ ¬DoorOpen(Door1, S)
    (You can open the door only if it is not already open.)
  • Effect: DoorOpen(Door1, do(Open(Door1), S))
    (After the action, the door becomes open.)

Advantages:

  • Captures dynamic changes logically.
  • Enables reasoning about what actions are possible.
  • Forms the foundation for automated planning.

Applications:
 Robotics, self-driving systems, intelligent agents, dynamic reasoning environments.

Question 4: Theorem Proving in First-Order Logic — Forward and Backward Chaining

Definition:
 Theorem proving is the process of using logical inference rules to determine whether a goal (theorem) can be logically derived from known facts and rules in a knowledge base.

1. Components of Theorem Proving

  • Knowledge Base (KB): Set of facts and rules.
  • Goal Statement: The conclusion we want to prove.
  • Inference Mechanism: Logical reasoning system (e.g., Modus Ponens, Unification).

2. Forward Chaining (Data-Driven Reasoning)

  • Starts with known facts and applies rules to derive new facts until the goal is reached.
  • Works from cause to effect.
  • Used in expert systems and production systems.

Process:

  1. Begin with initial facts in the KB.
  2. Find rules whose premises match current facts.
  3. Infer new facts.
  4. Repeat until the goal is derived or no more facts can be inferred.

Example:

  • Rule: If it rains → the ground gets wet.
  • Fact: It is raining.
  • Derived: The ground is wet.

Representation in FOL:

Rain(x) → WetGround(x)

Rain(Today)

⊢ WetGround(Today)

✅ Conclusion: WetGround(Today) is proved.

3. Backward Chaining (Goal-Driven Reasoning)

  • Starts from the goal and works backward to determine what facts must be true.
  • Used in Prolog and rule-based systems.

Process:

  1. Start with the goal.
  2. Find rules whose conclusion matches the goal.
  3. Convert rule premises into new sub-goals.
  4. Continue recursively until all sub-goals are proved.

Example:

  • Goal: Prove WetGround(Today).
  • Rule: Rain(x) → WetGround(x).
  • Sub-goal: Rain(Today).
  • Fact: Rain(Today) is true.
    ✅ Therefore, WetGround(Today) is true.

Question 5: Partial Order Planning vs Linear Planning with Example

1. Planning in AI

Planning means creating a sequence of actions to achieve a specific goal from an initial state.

2. Partial Order Planning (POP)

  • Definition:
     A non-linear planning approach where only necessary action orderings are specified.
    Independent actions can occur in     any order or in parallel.
  • Also called “least-commitment planning” because it delays order decisions until needed.

Features:

  • Not all actions are strictly ordered.
  • Allows parallel execution.
  • Efficient for complex and dynamic systems.
  • Used in modern planners like UCPOP.

Example:
 Goal: Pack for a trip
Actions:

  1. Pack clothes
  2. Pack toiletries
  3. Close suitcase

Dependencies:

  • “Close suitcase” must happen after both packing actions.
  • Packing actions can occur in any order.

Partial Order Plan:

Pack clothes → Close suitcase  

Pack toiletries → Close suitcase

Possible Linear Orders:

  1. Pack clothes → Pack toiletries → Close suitcase
  2. Pack toiletries → Pack clothes → Close suitcase

✅ Both valid. Parallel possible.

3. Linear Planning

  • Definition:
     A totally ordered sequence of actions where each step follows another in strict order.
  • Used in: Early AI planners (e.g., STRIPS).

Features:

  • Simple and deterministic.
  • Executes one action at a time.
  • Inefficient in complex domains.

Example:
 Goal: Make tea
Steps:

  1. Boil water
  2. Add tea leaves
  3. Pour tea into cup

Each step depends on the previous one.
Failure in one step halts the entire plan.

Question 6: Bayesian Networks — Representation and Reasoning Under Uncertainty

Definition:
 A Bayesian Network (BN) is a probabilistic graphical model that represents random variables and their conditional dependencies using a directed acyclic graph (DAG).

Used to perform reasoning under uncertainty — where outcomes are probabilistic, not deterministic.

1. Structure of a Bayesian Network

  • Nodes: Represent random variables (e.g., Rain, Sprinkler, Wet Grass).
  • Edges: Represent conditional dependencies (cause-effect relationships).
  • Conditional Probability Table (CPT): Quantifies how likely a node is given its parent nodes.

Graph Example:

Meaning:

Question 7: Forms of Learning in Artificial Intelligence with Examples

Definition:
 Learning in Artificial Intelligence (AI) refers to the ability of a system to improve its performance automatically through experience or data.
It enables AI to adapt, predict, and make decisions without explicit programming.

1. Supervised Learning

  • Concept:
     The system learns from a labeled dataset — both input and correct output are provided.
  • Goal: Predict outcomes for new, unseen data.
  • Common Algorithms: Linear Regression, Decision Trees, SVM, Neural Networks.
  • Examples:
  • Predicting house prices based on area, location, etc.
  • Email spam detection (input: email text → output: spam/not spam).
  • Used for: Classification and Regression problems.

2. Unsupervised Learning

  • Concept:
     Learns patterns or structure from unlabeled data (no predefined output).
  • Goal: Discover hidden patterns or groupings.
  • Common Algorithms: K-Means, Hierarchical Clustering, PCA.
  • Examples:
  • Customer segmentation in marketing.
  • Grouping similar news articles automatically.
  • Used for: Pattern discovery and clustering.

3. Semi-Supervised Learning

  • Concept:
     Uses both labeled and unlabeled data to improve learning accuracy.
  • Example:
  • Classifying medical images where only some are labeled.
  • Speech recognition systems.
  • Advantage:
  • Reduces data labeling cost.
  • Improves learning performance with limited labeled data.

4. Reinforcement Learning

  • Concept:
     Agent learns by interacting with the environment, receiving rewards or penalties for actions.
  • Goal: Maximize total reward.
  • Components:
  • Agent
  • Environment
  • Actions
  • Reward signal
  • Examples:
  • Training robots to walk.
  • Game-playing AI (Chess, AlphaGo).
  • Self-driving cars.
  • Used for: Sequential decision-making tasks.

5. Learning by Observation / Explanation-Based Learning

  • Concept:
     The system learns by observing human actions or analyzing examples and forming rules.
  • Examples:
  • Robot learns to make tea by watching a human.
  • Expert systems learn diagnostic rules from doctors.
  • Used for: Imitation learning and expert system development.

Question 8: Working of a Decision Tree in Supervised Learning — Strength and Limitation

Definition:
 A Decision Tree is a tree-structured model used in supervised learning for classification and regression.
It splits data based on the best attributes to predict an outcome.

Structure of a Decision Tree

  • Root Node: Represents the entire dataset and best attribute to split.
  • Internal Nodes: Test conditions on attributes.
  • Branches: Outcomes of tests.
  • Leaf Nodes: Final decision or class label.

Working (Step-by-Step)

  1. Select Best Attribute:
     Choose the attribute with the highest Information Gain (ID3), Gain Ratio (C4.5), or Gini Index (CART).
  2. Split the Data:
     Divide dataset into subsets based on the attribute’s values.
  3. Create Branches:
     Each subset becomes a branch.
  4. Repeat Recursively:
     Continue splitting until all records belong to one class or no attributes remain.
  5. Stopping Condition:
     Stop when nodes are pure or splitting no longer improves accuracy.

Strengths:

  1. Easy to understand and visualize.
  2. Handles both numeric and categorical data.
  3. No need for data scaling or normalization.
  4. Captures non-linear patterns.
  5. Fast and efficient on large datasets.
  6. Works for both classification and regression.

Limitations:

  1. Overfitting: May fit training data too closely.
  2. Instability: Small data changes can alter the tree.
  3. Bias: Toward attributes with many levels.
  4. Weak for continuous outputs: Single trees less accurate for regression.
  5. Greedy nature: May miss globally optimal tree.

Overcoming Limitations:

  • Pruning: Reduces tree complexity.
  • Ensemble Methods: Random Forest, Gradient Boosting.

Applications:

  • Medical diagnosis
  • Credit risk analysis
  • Weather forecasting
  • Spam detection
  • Customer churn prediction

Question 10: Unsupervised Learning and Application of Clustering with Example

Definition:
 Unsupervised Learning is a type of machine learning where the model learns patterns from unlabeled data — meaning no predefined outputs or categories are given.
The goal is to discover structure, similarity, or hidden relationships in the data automatically.

Characteristics:

  • Works on unlabeled datasets.
  • Focuses on pattern discovery and grouping.
  • Uses distance/similarity measures (e.g., Euclidean distance).
  • No supervision or target labels.

Main Types of Unsupervised Learning:

  1. Clustering → Grouping similar data points together.
  2. Association → Finding relationships between variables (e.g., Market Basket Analysis).
  3. Dimensionality Reduction → Reducing number of features (e.g., PCA).

Clustering in Unsupervised Learning

Definition:
 Clustering is the process of dividing a dataset into groups (clusters) so that:

  • Data points within the same cluster are similar.
  • Data points across clusters are dissimilar.

Steps in Clustering:

  1. Collect Data: Gather unlabeled data.
  2. Select Similarity Measure: Usually Euclidean distance or cosine similarity.
  3. Choose Algorithm: e.g., K-Means, Hierarchical, DBSCAN.
  4. Form Clusters: Group data based on similarity.
  5. Analyze and Label Clusters: Interpret discovered patterns.

Example: K-Means Clustering

Question 11: Market Basket Analysis and Its Role in Business Applications

Definition:
 Market Basket Analysis (MBA) is an unsupervised learning and data mining technique used to find relationships between items that customers frequently purchase together.
It helps in association rule mining — discovering patterns like “If X is bought, Y is also likely to be bought.”

Purpose:

To understand customer purchasing behavior and support:

  • Product placement
  • Cross-selling
  • Recommendations
  • Inventory management

Basic Principle:

If many customers buy item A and item B together, then there is a strong association rule between them:

A⇒BA \Rightarrow BA⇒B

This rule means: “If A is purchased, B is likely to be purchased.”

Question 12: Structure of a Neural Network — Role of Weights and Activation Functions

Definition:
 A Neural Network (NN) is a computational model inspired by the human brain, made up of layers of interconnected neurons (nodes).
It learns to recognize complex patterns through training.

Basic Structure:

A Neural Network has three main layers:

  1. Input Layer
  • Receives raw data (features).
  • Each neuron = one feature.
  • Example: For predicting house price → features = area, rooms, location.
  1. Hidden Layer(s)
  • Perform computations using weighted connections from previous layers.
  • Extracts important patterns and representations.
  1. Output Layer
  • Produces the final result (classification or prediction).
  • Example: Spam/Not Spam, Price value, etc.

Working:

  1. Each neuron receives inputs x1,x2,...,xnx_1, x_2, ..., x_nx1​,x2​,...,xn​.
  2. Each input is multiplied by a weight (w).
  3. The neuron computes a weighted sum:
    z=(w1x1+w2x2+...+wnxn)+bz = (w_1x_1 + w_2x_2 + ... + w_nx_n) + bz=(w1​x1​+w2​x2​+...+wn​xn​)+b
  4. The result zzz passes through an activation function to introduce non-linearity.
  5. The output propagates forward; errors are minimized using backpropagation (updating weights).

Role of Weights:

  • Control strength of connections between neurons.
  • Determine influence of each input feature.
  • Adjusted during training to minimize prediction error.
  • High weight → strong impact; Low weight → weak impact.

Example:
 If “study hours” has a higher weight than “sleep hours,” it influences “exam score” more.

Role of Activation Functions:

  • Decide whether a neuron should be activated.
  • Introduce non-linearity so the network can model complex data.
